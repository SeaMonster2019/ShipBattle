## 一、游戏概述

**虚空海战（Void Combat）** 是一款运行于浏览器的纯前端单机游戏，使用 HTML5 Canvas + 原生 JavaScript 实现，无需任何框架或后端。

本游戏的战斗机制高度还原自 Owlcat Games 制作的 CRPG《战锤40,000：行商浪人》（*Warhammer 40,000: Rogue Trader*）中的虚空舰船战斗（Void Combat）子系统，而该系统本身又源于 Fantasy Flight Games 出版的桌游规则书及《Battlefleet Gothic》舰队战棋的设计传统。

### 游戏特点

| 特点 | 说明 |
|------|------|
| 回合制战斗 | 玩家与敌方 AI 交替行动，策略性强 |
| 物理惯性移动 | 必须用完全部移动格数，不能随意停止 |
| 转向限制 | 走够 N 格才能转向一次，还原大型战舰的操控手感 |
| 移动中途开火 | 在规划路径时预标记开火时机，走到该位置时自动发射 |
| 四方向护盾 | 船头/左舷/右舷/背部护盾独立，战斗中不自动恢复 |
| 射击弧限制 | 每门炮只能攻击特定方向范围内的目标 |
| 岗位技能系统 | 6 种主动技能，涵盖机动、防御、特殊攻击 |
| 相机跟随 | 镜头自动平滑跟随双船中点，地图无限延展不会丢失视角 |

## 四、核心玩法规则（超详细）

### 4.1 世界地图与坐标系

游戏世界是一个 **40列 × 30行** 的方格海洋地图，每格像素大小为 52×52 px。

- **格坐标系**：以左上角为 (0,0)，向右为 col+，向下为 row+
- **世界像素坐标**：格 (col, row) 的中心像素 = (col×52 + 26, row×52 + 26)
- **8方向移动**：舰船可以沿 0°、45°、90°、135°、180°、225°、270°、315° 共 8 个方向移动

```
方向索引对照表：
  5(左上) 6(上) 7(右上)
  4(左)        0(右)
  3(左下) 2(下) 1(右下)
```

### 4.2 回合结构

每个完整回合由以下阶段组成：

```
【玩家回合】
  Phase 1：规划移动路径
    └─ 在格子地图上逐格点击，规划本回合的移动路线
    └─ 可在规划中途标记开火指令
    └─ 可使用技能（新航向、急转弯）修改路径能力
  Phase 2：确认路径 → 执行动画
    └─ 舰船按路径逐格移动
    └─ 途中经过标记的开火点时自动发射
  Phase 3：（可选）非规划时直接开火
    └─ 规划外，若武器已冷却且敌舰在弧内，可直接点击开火

【敌方回合】（自动进行）
  └─ AI 计算理想航向（试图将侧舷朝向玩家）
  └─ AI 按相同移动规则（机动性限制）移动
  └─ AI 到达终点后开火

【回合结算】
  └─ 双方武器冷却递减 1
  └─ 技能冷却递减 1
  └─ 回合计数 +1
```

> ⚠️ **重要**：护盾在回合结束时**不会自动恢复**。只能通过技能主动恢复。

---

### 4.3 移动规划系统

移动规划是本游戏最核心的机制，完整流程如下：

#### 第一步：开始规划

点击底部「📐 规划移动」按钮，进入规划模式：

- 当前舰船位置以**橙色边框**高亮显示
- 周围 8 个相邻格以**绿色（可移动）** 或 **红色（不可转向/越界）** 高亮
- 顶部显示「已走 0/5 格」进度

#### 第二步：点击格子延伸路径

依次点击相邻格子，路径以**绿色填充 + 虚线连接**显示。

**路径规则约束：**

| 约束 | 说明 |
|------|------|
| 只能点击**相邻格**（8方向） | 不能跳格 |
| **必须走完全部格数** | 速度=5 就必须规划出 5 格路径 |
| 转向需满足机动性条件 | 见 4.4 节 |
| 不能越出地图边界 | col < 0 或 col ≥ 40 等均无效 |

#### 第三步：标记中途开火（可选）

在规划中途的任意位置，点击开火按钮（「🔥 船头炮」等），系统会记录：

```
pendingFires = [
  { wep: 'PORT',      afterStepIdx: 2 },  // 走完第2步后发射左舷炮
  { wep: 'STARBOARD', afterStepIdx: 4 },  // 走完第4步后发射右舷炮
]
```

- **每门炮本回合只能标记一次**（已标记的按钮自动变灰）
- 执行时系统会**自动判断**当时位置和朝向是否满足射击弧要求，不满足则跳过

#### 第四步：确认路径

路径走满全部格数后，点击「✓ 确认路径」（或按 Enter）：
- 进入 `anim` 阶段
- 舰船按队列逐步移动，途中遇到开火标记自动开炮
- 最终进入敌方回合

#### 撤回步骤

规划时可以**右键点击画布**撤回最后一步，系统会重新计算机动性计数器。

### 4.4 转向机制

转向机制是本游戏还原度最高的规则，直接模拟大型战舰的惯性。

#### 核心规则

```
每次转向前，必须沿当前方向直线移动至少 Manoeuvrability 格
每次转向幅度只能是 45°（一个方向索引单位）
```

**示例（玩家舰，机动性 = 2）：**

```
当前朝向：→（方向0）
步骤 1：→ 前进，stepsSinceLastTurn = 1（不够 2，不能转）
步骤 2：→ 前进，stepsSinceLastTurn = 2（满足条件，可以转了）
步骤 3：↘ 转向，stepsSinceLastTurn 重置为 0
步骤 4：↘ 前进，stepsSinceLastTurn = 1
步骤 5：↘ 前进，stepsSinceLastTurn = 2（可以再次转向）
```

**转向角度对照：**

```
合法转向（±1 方向索引 = ±45°）：
  0(右) → 1(右下) ✓
  0(右) → 7(右上) ✓
  0(右) → 2(下)   ✗  超过45°，非法！
```

#### 技能对转向的影响

| 技能 | 效果 |
|------|------|
| **新航向** | 下一次转向允许转 90°（方向索引差 = 2） |
| **急转弯** | 立即将当前规划方向反转 180°，stepsSinceLastTurn 重置 |

### 4.5 武器系统与射击弧

每艘舰船共有 **4 门武器**，安装在不同位置，覆盖不同方向：

#### 武器参数表

| 武器 | 简称 | 方向 | 射击弧（半角） | 射程 | 伤害（随机） | 冷却 |
|------|------|------|----------------|------|-------------|------|
| **船头炮（Prow）** | F | 正前方 | ±45° | 6格 | 14~22 | 2回合 |
| **左舷炮（Port）** | P | 左侧 | ±60° | 5格 | 20~32 | 1回合 |
| **右舷炮（Starboard）** | S | 右侧 | ±60° | 5格 | 20~32 | 1回合 |
| **背部炮（Dorsal）** | D | 背后 | ±135° | 4格 | 12~18 | 1回合 |

> 射程单位：格，实际像素 = 格数 × 52

#### 射击弧判断逻辑

```
1. 计算敌舰相对于我舰的像素方向角（Math.atan2）
2. 计算武器的绝对方向角 = 船头方向角 + 武器偏移
   偏移：PROW=0°, PORT=-90°, STARBOARD=+90°, DORSAL=+180°
3. 计算角度差（规范化到 -180°~+180°）
4. 若 |角度差| ≤ 武器半弧角 → 在射击弧内 ✓
```

#### 射击弧可视化

非规划模式下，画布上会以**扇形区域**显示各武器的射程和弧度：

- 🟡 **黄色扇形** → 船头炮（PROW）
- 🔵 **蓝色扇形** → 左舷炮（PORT）
- 🟣 **紫色扇形** → 右舷炮（STARBOARD）
- 🟠 **橙色扇形** → 背部炮（DORSAL）

扇形边界颜色更深，如果敌舰在扇形内对应按钮亮起可用。

#### 开火方式

**方式一：规划中途预标记**（推荐，符合原作设计）

在路径规划阶段点击开火按钮，在当前路径步骤处插入一个开火事件。执行时舰船移动到对应位置后，自动判断射击弧并开火。

**方式二：即时开火**（规划结束后）

不进入规划模式时，若武器已冷却且敌舰在射击弧内，可直接点击按钮即时开炮。

### 4.6 伤害计算流程

每次炮击命中时，按以下顺序计算伤害：

```
基础伤害（BaseDmg）
    ↓ × (1 + scanBonus/100)   ← 弱点扫描加成
最终伤害（FinalDmg）
    ↓
判断命中的护盾面（F/P/S/D）
  └─ 根据"炮弹来袭方向" vs "目标船头方向"的相对角度判定
    ↓
强化护盾检查：若该面被强化 → FinalDmg × 0.5（减半）
    ↓
护盾吸收：shieldAbs = min(currentShield, FinalDmg)
  └─ 对应面护盾值 -= shieldAbs
    ↓
穿透伤害 = FinalDmg - shieldAbs
    ↓
护甲减免：armourBlock = min(penetrate, armour)
  └─ 默认护甲值：玩家=10，敌方=8
    ↓
实际船体伤害 = penetrate - armourBlock
    ↓
hull -= 实际船体伤害
  └─ hull ≤ 0 → 舰船沉没
```

**护盾面判定规则（防御方视角）：**

```
攻击来自相对防御方船头的方向 diff = normDir(fromDir - ship.dir)：

diff = 0, 1, 7  → 船头面（F）被击中
diff = 2, 3     → 右舷面（S）被击中
diff = 4, 5     → 背部面（D）被击中
diff = 6        → 左舷面（P）被击中
```

### 4.7 护盾系统

护盾是承受伤害的第一道防线，运作方式如下：

#### 基本规则

- 护盾分为四个独立扇区：**F（船头）、P（左舷）、S（右舷）、D（背部）**
- 各扇区护盾值**互不影响**，某方向护盾归零不影响其他方向
- **战斗中护盾不自动恢复**，必须通过技能主动回复
- 护盾归零后，该方向来袭的伤害将直接进入护甲/船体计算

#### 护盾容量

| 舰船 | 船头F | 左舷P | 右舷S | 背部D |
|------|-------|-------|-------|-------|
| 玩家（圣裁号） | 40 | 55 | 55 | 30 |
| 敌方（异端号） | 35 | 45 | 45 | 25 |

#### 护盾可视化

护盾以**彩色弧线**显示在舰船周围，亮度 = 当前护盾比例：

```
绿色弧线（船头方向）= F 护盾
蓝色弧线（左侧）    = P 护盾
紫色弧线（右侧）    = S 护盾
橙色弧线（背后方向）= D 护盾
```

护盾越低弧线越暗，归零则弧线消失。

#### 护盾恢复技能

| 技能 | 效果 | 代价 |
|------|------|------|
| **强化护盾** | 指定某一方向减半下次受击，弧线变白色虚线 | CD=2 |
| **重启护盾** | 全部四方向护盾立即满额恢复 | CD=5，下回合速度-1 |

### 4.8 岗位技能系统

共有 **6 个主动技能**，对应原作中的 6 个舰船岗位。所有技能均有冷却时间（CD），以回合为单位。

---

#### ① 新航向（New Heading）
**岗位**：机动主官（Master of Manoeuvres）

```
效果：下一次转向允许一次性转动 90°（正常上限为 45°）
使用时机：必须在规划模式下使用
冷却：3 回合
限制：每次规划只能激活一次（newHeadingUsed 标记）
```

**适用场景**：当你需要快速调整侧舷朝向，而剩余格数不足以通过两次 45° 转向实现时。

---

#### ② 急转弯（Swing Run）
**岗位**：最高指挥官（Supreme Commander）

```
效果：立即将当前规划方向反转 180°，方向指针 = normDir(dir + 4)
      转向后 stepsSinceLastTurn 重置为 0
使用时机：必须在规划模式下使用，且已走至少 1 步（加速阶段）
冷却：4 回合
限制：每次规划只能使用一次（swingRunUsed 标记）
```

**适用场景**：当你想原地掉头逃跑，或让背部炮对准敌方。执行时路径中会留下掉头标记，动画时会有明显的转向动作。

---

#### ③ 强化护盾（Reinforce Shields）
**岗位**：护盾主官（Shield Master）

```
效果：选择 F/P/S/D 中一个护盾面，该面下次受击伤害减半（×0.5）
      使用后该方向出现白色虚线弧标记
      触发一次后标记消除
使用时机：任意（规划外）
冷却：2 回合
```

**使用方法**：点击按钮后弹出输入框，输入 F/P/S/D 确认（大小写均可）。

**适用场景**：预判敌方下回合会攻击某方向时，提前强化对应护盾。

---

#### ④ 重启护盾（Restart Shields）
**岗位**：护盾主官 · 高级能力

```
效果：全部四方向护盾（F/P/S/D）立即恢复至满值
代价：下一回合舰船速度 -1（restartPenalty 标记）
使用时机：任意（规划外）
冷却：5 回合
```

**适用场景**：护盾全面告急、准备迎接敌方大规模炮击时使用。代价是下回合机动受限。

---

#### ⑤ 翘曲波（Warp Wave）
**岗位**：翘曲导能师（Warp Channeler）

```
效果：强制敌舰随机旋转 45°（顺时针或逆时针）
      若敌舰已有船体损伤，额外造成 缺失船体 × 15% 的纯粹伤害
使用时机：任意（规划外）
冷却：4 回合
射程：无限制
```

**适用场景**：
- 破坏敌方的射击角度，使其侧舷炮偏离玩家
- 对受损敌舰使用时有可观的额外伤害（满血=0伤，50%血量=7.5%最大血量的伤害）

---

#### ⑥ 弱点扫描（Vulnerability Scan）
**岗位**：以太主官（Master of Etherics）

```
效果：标记敌舰弱点，下一次受击伤害 +50%（scanBonus = 50）
      命中后标记消除
使用时机：任意（规划外）
冷却：3 回合
```

**使用节奏**：先扫描 → 当回合或下回合用侧舷炮重伤打击，效果翻倍级别。

### 4.9 敌方 AI 行为

敌方舰船（异端号）拥有简单但有效的 AI 策略：

#### 移动策略

```
1. 计算玩家相对敌舰的方向角
2. 计算两种理想船头方向：
   - idealDir1 = 玩家方向 - 2（使右舷朝向玩家）
   - idealDir2 = 玩家方向 + 2（使左舷朝向玩家）
3. 选择与当前方向差值更小的理想方向
4. 按机动性限制（每 3 格转一次 45°）逐步向理想方向调整
5. 遇边界时停止前进（不转向）
```

#### 开火策略

```
移动结束后，按以下优先级尝试开火：
  PORT（左舷炮）→ STARBOARD（右舷炮）→ PROW（船头炮）→ DORSAL（背部炮）
  首先满足射击弧和冷却条件的武器开火，仅开一次
```

#### AI 属性

| 属性 | 数值 |
|------|------|
| 速度（Speed） | 4格/回合 |
| 机动性（Mano） | 3格/转 |
| 船体（Hull） | 100 |
| 护甲（Armour） | 8 |

> AI 受到和玩家**完全相同的移动规则约束**（机动性限制、边界检测等），并非作弊。

## 六、战术指南

### 基础原则

#### ① 先规划移动，边走边打

这是最符合原作设计的玩法。开始规划时，先想好终点位置（能让侧舷对准敌人），再在路径途中标记开火时机。

```
正确节奏示例：
  步骤1 → 步骤2 → 🔥标记左舷炮 → 步骤3 → 步骤4 → 步骤5（终点）
  执行：移动2步后开左舷炮，继续移动3步到达终点
```

#### ② 侧舷火力最强，优先让侧舷对准敌舰

- 左舷炮/右舷炮：伤害 20~32，冷却仅 1 回合
- 船头炮：伤害 14~22，冷却 2 回合（但射程最远）
- 背部炮：伤害最低（12~18），但射击弧最宽（±135°），紧急时候用

#### ③ 护盾是战术资源，不是免费防御

- 护盾不恢复，每次被击中都是永久损耗（除非用技能）
- 调整朝向让**高护盾的侧舷**对准敌人的主要火力方向
- 在护盾告急前使用「重启护盾」，而不是等到归零后才用

#### ④ 提前读敌方意图

AI 总是试图把侧舷对准你：
- 如果敌舰在你的左边，它会尝试让自己的右舷朝向你
- 你可以预判它的终点位置，提前调整朝向让后方或侧面对准它
- 使用「翘曲波」强制旋转敌舰，打乱其射击角度

#### ⑤ 技能组合打法

**高爆发连击：**
```
弱点扫描 → 下回合侧舷炮命中 → 实际伤害 ×1.5
配合翘曲波先破坏敌方护盾朝向，效果更佳
```

**防御型打法：**
```
强化护盾（预判受击方向）→ 重启护盾（护盾全满）→ 稳健交换血量
```

**急转弯奇袭：**
```
普通行进 → 急转弯 → 背部炮对准之前在正前方的敌舰
配合新航向快速调整到最佳射击角度
```

### 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 忘记路径必须走满 | 无法确认路径 | 提前规划好终点 |
| 转向太急（走的格不够） | 规划被拒绝 | 看顶部「再走N格可转向」提示 |
| 开火时敌舰不在弧内 | 炮击无效跳过 | 移动到合适位置再标记开火 |
| 护盾耗尽后不恢复 | 船体被直接打穿 | 及时使用重启护盾 |
| 重启护盾后下回合机动受限 | 无法走足格数 | 速度-1 意味着必须路径更短 |

### 7.5 渲染管线

每帧调用 `renderAll()`，渲染层次如下：

```
renderAll()
  │
  ├─ ctx.clearRect(全画布)
  ├─ 填充海面底色（深蓝黑 #07121f）
  │
  ├─ ctx.save()

  │   │
  │   ├─ 波浪线（10条 sin 曲线，waveOffset 驱动动画）
  │   ├─ 格子网格（仅绘制视口内可见格）
  │   │

  │   ├─ [规划中] drawPlanPath()     ← 规划路径高亮
  │   │   └─ drawFireMarkers()       ← 路径中的开火标记图标
  │   │
  │   ├─ Particle.draw()             ← 爆炸/尾流粒子
  │   ├─ Projectile.draw()           ← 飞行中的炮弹
  │   ├─ Ship.draw() (enemy)         ← 敌舰（先画，被玩家覆盖）
  │   └─ Ship.draw() (player)        ← 玩家舰
  │       ├─ 船体多边形（线性渐变）

  │
  └─ drawScreenHUD()                 ← 屏幕固定HUD（路径进度文字等）
```

#### 性能优化

- **格子裁剪**：只绘制 `cam.x/TILE ~ (cam.x+canvasW)/TILE` 范围内的格子
- **粒子生命周期管理**：`life <= 0` 时从数组删除
- **lerp 插值**：相机和船只位置用线性插值，避免逐帧抖动

### 7.6 相机系统

```javascript
// 相机状态
const cam = { x: 0, y: 0, targetX: 0, targetY: 0 };

// 每帧：计算目标位置（双船中点居中）
function updateCamera() {
    const midX = (player.animX + enemy.animX) / 2;
    const midY = (player.animY + enemy.animY) / 2;
    cam.targetX = midX - canvas.width  / 2;
    cam.targetY = midY - canvas.height / 2;
    // 边界夹紧，防止显示地图外空白
    cam.targetX = clamp(cam.targetX, 0, max(0, WORLD_COLS*TILE - canvas.width));
    cam.targetY = clamp(cam.targetY, 0, max(0, WORLD_ROWS*TILE - canvas.height));
}

// 每帧：平滑插值
function lerpCamera(dt) {
    cam.x = lerp(cam.x, cam.targetX, dt * 6);  // 系数6 ≈ 约0.15秒跟上
    cam.y = lerp(cam.y, cam.targetY, dt * 6);
}

// 渲染时应用
ctx.translate(-Math.round(cam.x), -Math.round(cam.y));
```

相机始终以**双船中点**为目标，平滑跟随；整数对齐（`Math.round`）防止次像素渲染造成的抖动。

#### 屏幕坐标与世界坐标转换

```javascript
// 鼠标点击事件：屏幕坐标 → 世界坐标 → 格坐标
canvas.addEventListener('click', e => {
    const rect = canvas.getBoundingClientRect();
    const sx = e.clientX - rect.left;
    const sy = e.clientY - rect.top;
    const wx = sx + cam.x;   // 屏幕坐标 + 相机偏移 = 世界像素坐标
    const wy = sy + cam.y;
    const col = Math.floor(wx / TILE);
    const row = Math.floor(wy / TILE);
    tryAddStep(col, row);
});
```

### 7.7 动画队列系统

游戏采用**异步分步动画队列**，而非实时物理模拟，保证每步都有视觉反馈。

#### 执行原理

```
executePlan() 构建队列:
  [move→5,14] [fire→PORT] [move→6,14] [move→7,13] [move→8,13] [move→9,13] [enemyTurn]

processAnimQueue(dt) 每帧调用:
  animStepTimer += dt
  当 animStepTimer >= 该步骤的时间阈值:
    执行当前步骤逻辑（更新 ship.col/row/dir，发射炮弹）
    action.done = true
    animIdx++，处理下一步
```

#### 各步骤时间阈值

| 步骤类型 | 时间阈值 | 说明 |
|----------|----------|------|
| `move` | 0.12 秒 | 每格移动约 120ms，5格合计约 0.6秒 |
| `swingrun` | 0.20 秒 | 急转弯稍有停顿感 |
| `fire` | 0.08 秒 | 几乎即时发射 |
| `enemyTurn` | 0.30 秒 | 敌方行动前的短暂延迟 |

#### 炮弹与队列的解耦

炮弹（`Projectile`）的飞行是**独立于队列的实时物理**，不阻塞队列推进。命中检测通过回调函数 `onHit()` 实现：

```javascript
new Projectile(sx, sy, tx, ty, color, () => {
    // 炮弹到达目标时自动调用
    target.takeDamage(dmg, fromDir);
    spawnExplosion(tx, ty);
    updateHUD();
    checkGameOver();
});
```

### 7.8 AI 实现

AI 在 `runEnemyAI()` 中完整执行，无异步延迟（由 `enemyTurn` 队列步骤触发）：

```javascript
function runEnemyAI() {
    // 1. 计算理想船头方向（侧舷对玩家）
    const toPlayer = atan2(dy, dx)  // 玩家方向
    idealDir = 使右舷或左舷朝向玩家的船头方向（选角度差更小的）

    // 2. 模拟移动（受机动性约束）
    for step in range(enemy.speed):
        if 可以转向 and 需要转向:
            朝理想方向转一步（45°）
            stepsSinceLastTurn = 0
        前进一格（若不越界）
        stepsSinceLastTurn++

    // 3. 更新敌舰位置和方向
    enemy.col = curCol; enemy.row = curRow; enemy.dir = curDir

    // 4. 冷却递减
    enemy.tickCooldowns()

    // 5. 开火（优先级：PORT > STARBOARD > PROW > DORSAL）
    for wep in [PORT, STARBOARD, PROW, DORSAL]:
        if checkFiringArc(enemy, player, wep) and wepCd == 0:
            doEnemyFire(wep)
            break
}
```
